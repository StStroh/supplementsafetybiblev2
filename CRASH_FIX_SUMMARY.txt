BUG: NotFoundError: Failed to execute 'insertBefore' on 'Node'
TRIGGER: Select "Losartan" + "Potassium Chloride" → Click "Run Check" → CRASH
STATUS: ✅ FIXED

================================================================================
ROOT CAUSE
================================================================================

Unstable React keys (key={idx}) in citation rendering caused DOM reconciliation
failures. When citations array changed during re-render, React couldn't track
which DOM nodes to update → insertBefore() error.

================================================================================
TECHNICAL DETAILS
================================================================================

BEFORE (BROKEN):
{interaction.citations.map((citation: any, idx: number) => (
  <a key={idx} href={citation.url}>           ❌ UNSTABLE KEY
    • {citation.source}: {citation.title}
  </a>
))}

AFTER (FIXED):
{interaction.citations.map((citation: any, idx: number) => {
  const citationKey = citation.url || `${citation.source}-${citation.title}-${idx}`;
  return (
    <a key={citationKey} href={citation.url}>  ✅ STABLE KEY
      • {citation.source}: {citation.title}
    </a>
  );
})}

WHY IT WORKS:
- Stable keys let React correctly identify which nodes moved/changed
- No more DOM pointer confusion
- Clean reconciliation without insertBefore errors
